# Collaborating on Gaia

How **Dara** and **Cappytan** (and the AI agents) work on this repo without tripping over each
other. One page; for the full detail it points back to [`CLAUDE.md`](CLAUDE.md).

## The split: design vs. implementation

| | Owns | What that means |
|---|---|---|
| **Dara** | Classes, abilities, lore, art, the **REQUIEM canon** | Primary designer of class/ability/lore *design*. Authors the world. |
| **Cappytan + agents** | Gameplay *mechanics* — combat balance, level layout, loot scaling, the art *pipeline*, the TS engine | Implements Dara's designs; drives the numbers and the code. |

Agents **assist**: they implement Dara's designs, draft proposals for his review, fill genuine
gaps, and **flag conflicts for him to decide**. They never overrule canon. **Dara rules.**

The simplest rule that prevents most collisions: **don't edit each other's lane.** Want a class to
*play* differently? Change the design/canon. Want a fight *harder*? Tune the numbers.

## Where the lanes live on disk

| Dara's lane | Cappytan's / agents' lane |
|---|---|
| `docs/design/requiem/**` — the canon | `app/src/systems/**` — pure game logic |
| `assets/reference/**` — raw reference art | `app/src/controllers/**`, `app/src/ui/**` — flow + presentation |
| `DESIGN.md` — design intent | `app/src/data/enemies.ts`, `data/zones.ts` — balance/encounters |
| Class / ability / lore *decisions* | `app/tools/slice-art.py` + generated `app/assets/**` — art pipeline |

The **architecture layering** (ADR 0005) helps here too: `data ← systems ← controllers → ui`. Two
people working in different layers rarely produce merge conflicts. New content → `data/`; new pure
logic → `systems/` (with a test); DOM/flow → `controllers/`+`ui/`.

## The three collision traps

1. **Generated files — never hand-edit.** `data/requiem-kits.ts` is generated by
   `docs/design/requiem/gen-kits.cjs`, and the parsed REQUIEM files come from `parse-requiem.js`.
   Edit the **source** (`requiem-compendium.source.html` / the generator) and re-run — don't edit
   the output, or the next regen clobbers it.

2. **Canon vs. the POC slice.** The shipped game is an *invented placeholder* (a damage affinity
   ring + one signature effect per attunement; only the SOL classes are hand-tuned). REQUIEM is the
   truth. **When they disagree, REQUIEM wins** — but the move is to *surface* the conflict, not
   silently change either side. The `requiem-canon-keeper` agent flags drift; **Dara decides.**

3. **Git branches.** Work on separate feature branches; merge to `main` via PR. Don't push to each
   other's branches without permission. CI builds `dist/` to GitHub Pages on merge, so **`main`
   stays shippable** — keep `npm run typecheck` and `npm test` green before merging.

## Before you merge

- `npm run typecheck` — clean (`strict` is on).
- `npm test` — Vitest, the pure `systems/`.
- `npm run sim` — balance regression, if you touched gameplay numbers.
- Bump `GAME_VERSION` (`app/src/data/version.ts`) on a player-visible change; use the
  `vX.Y: summary` commit style.
- Record hard-to-reverse decisions as an ADR in `docs/adr/`.

When in doubt about who owns a call: **class/ability/lore design is Dara's; balance, levels, loot
scaling, art pipeline, and code are agent-driven.** Conflicts get *flagged*, not overruled.
